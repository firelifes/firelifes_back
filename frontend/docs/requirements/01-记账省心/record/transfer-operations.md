## 功能概述

将记账页面顶部 tab 从「支出 | 收入」扩展为「支出 | 收入 | 转账」。

**核心原则：转账操作不是收支。** 转账、还债、借贷都是资金在账户间的转移（底层统一为账户间转账），不应计入收支统计。此前转账和还债藏在支出分类里，既不直观又容易混淆统计。

### 行业依据
- **鲨鱼记账**：独立债务追踪模块，支持借出/借入、还款进度、逾期提醒
- MoneyWiz/MoneyThings/YNAB 等专业记账APP均将"非收支操作"独立为第三类
- 钱迹明确声明："转账、还款、借入、借出都只是单纯的资金转移，并没有产生具体的收支行为"
- CountAbout 是行业唯一实现全自动本金/利息拆分的APP

### 用户故事

| 序号 | 用户角色 | 场景 | 需求 | 价值 |
|------|---------|------|------|------|
| 1 | 普通用户 | 从储蓄卡转钱到微信 | 快速完成账户间资金转移 | 便捷 |
| 2 | 信用卡用户 | 每月还信用卡账单 | 一键还款，不影响收支统计 | 准确 |
| 3 | 房贷用户 | 还房贷时自动拆分本金利息 | 利息自动计入支出，本金冲减负债 | 专业 |
| 4 | 经常借钱给朋友的用户 | 记录借出和收回 | 自动追踪债权，不影响收支 | 清晰 |
| 5 | 有私人借款的用户 | 记录借入和偿还 | 自动追踪债务，不影响收支 | 清晰 |

## 各操作类型详细设计

### 1. 转账

**场景**：储蓄卡→微信、现金存银行、支付宝→银行卡

**账户变化**：
```
转出账户余额：↓
转入账户余额：↑
总资产：不变
总负债：不变
净值：不变
收支统计：不影响
```

**交互**：
```
┌─────────────────────────────────┐
│  🔄 转账                        │
│                                  │
│  从 [  储蓄卡  ▼ ]             │
│  到 [  微信零钱  ▼ ]           │
│                                  │
│         ¥ 0.00                   │
│                                  │
│  备注：点击填写                  │
│                                  │
│  [        确认转账       ]       │
└─────────────────────────────────┘
```

**账户选择规则**：
- 转出账户：全部账户可选
- 转入账户：全部账户可选，排除已选转出账户

**后端处理**：
- 生成1条 `type=transfer` 记录
- 转出账户余额减少，转入账户余额增加

---

### 2. 还信用卡

**场景**：用储蓄卡还信用卡账单

**账户变化**：
```
还款账户（资产）余额：↓
信用卡账户（负债）余额：↑（负得少了，欠款减少）
总资产：↓（还款账户减少）
总负债：↓（信用卡欠款减少）
净值：不变（资产减少 = 负债减少）
收支统计：不影响（刷卡消费时已记过支出）
```

**交互**：
```
┌─────────────────────────────────┐
│  💳 还信用卡                    │
│                                  │
│  从 [  储蓄卡  ▼ ]             │  ← 仅资产类账户
│  还 [  信用卡  ▼ ]             │  ← 仅信用卡类账户
│                                  │
│         ¥ 0.00                   │
│                                  │
│  备注：点击填写                  │
│                                  │
│  [        确认还款       ]       │
└─────────────────────────────────┘
```

**账户选择规则**：
- 还款账户：仅资产类（cash/investment/fixed_asset/depreciable_asset）
- 信用卡账户：仅信用卡类（credit_card）

**后端处理**：
- 生成1条 `type=repayment` 记录
- 还款账户余额减少，信用卡余额增加（负值变少）

**与原"还债"的区别**：
- 原来的"还债"统一处理所有负债，现在拆分为"还信用卡"和"还贷款"
- 信用卡还款不需要拆分本金/利息（刷卡消费时已记支出）
- 信用卡还款更简单：全额还款即可

---

### 3. 还贷款（⭐ 核心功能：本金+利息拆分）

**场景**：还房贷、车贷、消费贷等

**账户变化**：
```
还款账户（资产）余额：↓（扣除还款总额）
贷款账户（负债）余额：↓（仅本金部分减少）
利息支出：↑（自动生成一笔支出记录）
总资产：↓（还款账户减少）
总负债：↓（本金部分减少）
净值：↓（利息部分是真实支出）
收支统计：仅利息部分计入支出
```

**交互**：
```
┌─────────────────────────────────┐
│  🏠 还贷款                      │
│                                  │
│  从 [  储蓄卡  ▼ ]             │  ← 仅资产类账户
│  还 [  房贷-建行  ▼ ]          │  ← 仅负债类账户（liability）
│                                  │
│         ¥ 5,000.00              │
│                                  │
│  ── 本金利息拆分 ──             │
│                                  │
│  本金：[ ¥3,200  ▼ ]            │  ← 可编辑，自动计算利息
│  利息：[ ¥1,800  ▼ ]            │  ← 可编辑，自动计算本金
│                                  │
│  利息分类：[利息支出      ▼]    │  ← 利息记入哪个支出分类
│  备注：点击填写                  │
│                                  │
│  [        确认还款       ]       │
└─────────────────────────────────┘
```

**拆分逻辑**：
- 如果贷款账户已配置利率信息，自动计算当期利息（还款金额×月利率）和本金
- 如果贷款账户未配置利率信息，默认全部是本金（利息为0）
- 同时支持手动调整
- 调整任意一项，另一项自动同步，保持「本金+利息=总金额」
- 金额不允许为负数，且不能超过总金额

**后端处理**：
- 生成2条记录：
  - ① `type=repayment`，金额=本金部分 → 还款账户余额↓，贷款账户余额↓（本金减少）
  - ② `type=expense`，金额=利息部分 → 还款账户余额↓（再次减少），分类=用户选择的利息分类
- 两笔共享同一日期和备注（备注可追加"利息部分"标识）

### 4. 借出/收回

**场景**：借钱给朋友、垫付团建费用、收回借款

**借出 - 账户变化**：
```
现金账户余额：↓
应收账款（虚拟账户）余额：↑
总资产：不变（现金→债权，资产形态变化）
净值：不变
收支统计：不影响
```

**收回借款 - 账户变化**：
```
现金账户余额：↑
应收账款（虚拟账户）余额：↓
总资产：不变（债权→现金，资产形态变化）
净值：不变
收支统计：不影响
```

**交互**：
```
┌─────────────────────────────────┐
│  🤝 借出/收回                   │
│                                  │
│  ┌──────────┐ ┌──────────┐     │
│  │  借出    │ │  收回    │     │  ← 方向切换
│  └──────────┘ └──────────┘     │
│                                  │
│  对方：[  张三  ▼/输入 ]        │  ← 选择或新建联系人
│  从 [  储蓄卡  ▼ ]             │  ← 借出：资产类账户
│                                  │
│         ¥ 0.00                   │
│                                  │
│  备注：点击填写                  │
│                                  │
│  [        确认借出       ]       │
└─────────────────────────────────┘
```

**应收账款（隐式账户）**：
- 用户选择"借出给张三"时，系统自动创建/更新 `应收-张三` 虚拟账户
- 类型为 `receivable`（资产类），用户无需手动管理
- 在资产总览页的"债权"分组下展示
- 收回时自动抵减对应应收账款余额

**后端处理**：
- 借出：生成1条 `type=transfer` 记录，from=现金账户，to=应收账款
- 收回：生成1条 `type=transfer` 记录，from=应收账款，to=现金账户

---

### 5. 借入/偿还

**场景**：找朋友借钱、网贷到账、偿还私人借款

**借入 - 账户变化**：
```
现金账户余额：↑
应付账款（虚拟账户）余额：↑（债务增加）
总资产：↑
总负债：↑
净值：不变
收支统计：不影响
```

**偿还借款 - 账户变化**：
```
现金账户余额：↓
应付账款（虚拟账户）余额：↓（债务减少）
总资产：↓
总负债：↓
净值：不变
收支统计：不影响
```

**交互**：
```
┌─────────────────────────────────┐
│  📋 借入/偿还                   │
│                                  │
│  ┌──────────┐ ┌──────────┐     │
│  │  借入    │ │  偿还    │     │  ← 方向切换
│  └──────────┘ └──────────┘     │
│                                  │
│  对方：[  李四  ▼/输入 ]        │  ← 选择或新建联系人
│  到 [  储蓄卡  ▼ ]             │  ← 借入：资产类账户
│                                  │
│         ¥ 0.00                   │
│                                  │
│  备注：点击填写                  │
│                                  │
│  [        确认借入       ]       │
└─────────────────────────────────┘
```

**应付账款（隐式账户）**：
- 用户选择"从李四借入"时，系统自动创建/更新 `应付-李四` 虚拟账户
- 类型为 `payable`（负债类），用户无需手动管理
- 在资产总览页的"债务"分组下展示
- 偿还时自动抵减对应应付账款余额

**后端处理**：
- 借入：生成1条 `type=transfer` 记录，from=应付账款，to=现金账户
- 偿还：生成1条 `type=transfer` 记录，from=现金账户，to=应付账款

### 隐式账户展示规范

参考鲨鱼记账的债务追踪模块设计：

```
┌──────────────────────────────────┐
│         资产总览                 │
├──────────────────────────────────┤
│ 💰 现金资产    ¥10,000          │
│ 💼 投资资产    ¥50,000          │
│ 📄 债权        ¥2,000          │  ← 独立分组
│   ├─ 应收-张三   ¥1,500        │
│   └─ 应收-李四     ¥500        │
│ 🏠 固定资产    ¥500,000         │
│ ────────────────────────────────│
│ 💳 信用卡负债  -¥3,000          │
│ 📋 债务        ¥5,000          │  ← 独立分组
│   └─ 应付-王五   ¥5,000        │
```

**展示规则**：
- 隐式账户**不显示**在账户列表页（避免用户误操作）
- 在资产总览页的"债权"/"债务"分组下展示
- 明细页支持按"债权/债务"类型过滤

### 隐式账户结构

```typescript
interface ImplicitAccount {
  id: string                  // 账户ID
  name: string                // 格式：应收-张三 / 应付-李四
  type: 'receivable' | 'payable'
  counterparty: string        // 对方名称
  balance: number             // 当前余额
  isImplicit: true            // 标记为隐式账户
  isVisible: false            // 不在账户列表页显示（仅在资产总览债权/债务分组展示）
  createdAt: Date             // 创建时间
  updatedAt: Date             // 更新时间
}
```

## 明细页展示规则

转账/还债/借贷记录在明细页（BillCard）中展示，与账户金额调整记录同一层级：

- **显示位置**：按日期分组，在明细页记录列表中正常展示
- **显示前缀**：使用 `→` 符号，区别于收入的 `+` 和支出的 `-`
- **金额样式**：中性灰色（`--color-text-secondary`），与调整记录一致
- **默认名称**：
  - 转账 → "转账"
  - 还信用卡 → 使用备注，若无则显示 "还款"
  - 还贷款 → 使用备注，若无则显示 "还款"
  - 借出/借入 → 使用备注，若无则显示转账类型名称
- **收支统计**：不计入日/月/年收支汇总，不影响收入、支出、结余计算
- **分类修改**：不支持修改分类（转账记录无 typeId）

## 边界情况

1. **同账户转账**：转出和转入相同，提示禁止
2. **还贷款金额 < 利息**：提示"还款金额不足以覆盖当期利息"
3. **还贷款金额 > 月供**：多还部分全部冲减本金，提示"本期多还本金 ¥xxx，可缩短x期"
4. **无信用卡账户时选还信用卡**：提示"请先创建信用卡账户"
5. **无负债账户时选还贷款**：提示"请先创建贷款账户"
6. **借出/借入对方名重复**：同名视为同一人，复用已有隐式账户
7. **收回金额 > 应收余额**：提示"收回金额超过应收余额 ¥xxx"
8. **偿还金额 > 应付余额**：提示"偿还金额超过应付余额 ¥xxx"
9. **隐式账户删除**：应收/应付账户余额为0时可删除，有余额不可删除
10. **贴息处理**：还贷款时贴息金额计入本金部分（已由 LiabilityForm 支持）
11. **弹框高度溢出**：还贷款表单内容较多时，弹框高度限制在 80-85vh，内容区域可滚动，遮罩层可正常点击关闭

### 本金+利息拆分实时联动（v1.4 修复）

**问题**：用户在弹框键盘输入金额时，本金+利息拆分出现延迟，导致 `本金 + 利息 ≠ 用户输入的总金额`。

**根因**：`RepaymentSplitForm` 的 `watch(() => props.totalAmount, ...)` 条件为 `principal === 0 && interest.value === 0`，目的是"只在初次输入时自动拆分"。但这导致：
- 用户输入 "8" → totalAmount=8 → 拆分生效：principal=8, interest=0
- 用户继续输入 "9" → totalAmount=89 → 条件 `8 === 0 && 0 === 0` 不满足 → 拆分停留 principal=8, interest=0
- 拆分总和 = 8 ≠ 89 ❌

**修复**：
- 去掉 `principal === 0 && interest === 0` 条件
- 改为：只要 `totalAmount > 0` 就用最新金额重新拆分
- 拆分算法统一为：先算 `interest = calculateAutoInterest(newTotal)`，再算 `principal = newTotal - interest`，最后用 `Math.round(... * 100) / 100` 处理浮点
- 保障：`principal + interest = totalAmount` 始终成立（由算法决定，不依赖外部状态）

**用户故事**：作为还房贷的用户，**我想要** 在键盘连续输入金额时，本金/利息拆分能**实时同步**更新，**以便** 拆分的总和始终等于我输入的金额，无需手动调整。

**验收标准**：
- [x] 用户在键盘输入 "8" → 拆分：principal=8, interest=0
- [x] 用户继续输入 "9" 组成 "89" → 拆分立即更新：principal=89, interest=0
- [x] 用户清空金额（displayAmount=""）→ 拆分归零：principal=0, interest=0
- [x] 用户清空再输入新金额 → 拆分重新计算
- [x] 拆分永远满足 `principal + interest = totalAmount`（浮点误差由 Math.round 处理）
- [x] 用户手动编辑本金 → 利息自动联动（保持总和）
- [x] 用户手动编辑利息 → 本金自动联动（保持总和）
